# LX Z:IN 제품 이미지 자산 관리 규칙 (Asset Convention)

본 문서는 제품 이미지의 손실이나 경로 불일치를 방지하기 위한 강제 규칙을 정의합니다. 모든 개발자는 이미지 추가/수정 시 본 규칙을 반드시 준수해야 합니다.

## 1. 이미지 파일명 명명 규칙 (Naming Convention)

- **형식**: `{model_id}_{product_id}.jpg`
- **제한 사항**:
  - 공백, 특수문자 금지
  - **하이픈(`-`) 사용 금지**: 반드시 언더바(`_`)로 치환해야 함. (예: `FMKGA077-W0125` -> `FMKGA077_W0125`)
  - 모든 문자는 소문자가 아닌 원본 `model_id` 케이스를 유지하되 특수문자만 제거.
- **예시**: `FMKGA077_W01250000_13304.jpg`

## 2. 저장 위치

- **경로**: `/public/assets/lxzin/`
- 데이터베이스(`src/data/lxzin-products.js`)의 `imageUrl` 필드는 항상 `/assets/lxzin/파일명` 형식을 유지해야 함.

## 3. 정합성 유지 (Consistency)

- 제품 데이터를 업데이트하거나 새로 크롤링할 경우, 반드시 `verify-assets.cjs` 스크립트를 실행하여 모든 제품에 유효한 이미지 파일이 존재하는지 확인해야 합니다.
- 이미 존재하는 이미지를 덮어쓰지 않도록 주의하며, 파일 크기가 0이거나 1KB 미만인 경우 비정상적인 다운로드로 간주합니다.

## 4. 자동 검증 방법

```bash
node verify-assets.cjs
```

위 명령어를 실행하여 "Missing" 또는 "Invalid" 결과가 0이어야 배포 가능합니다.
