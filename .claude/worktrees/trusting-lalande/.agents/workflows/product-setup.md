# 📋 제품 작업 표준 절차 (Product Setup Workflow)

> **핵심 원칙**: 공식 사이트(lxzin.com)에서 확인한 데이터가 유일한 정답이다. 한 번에 모든 항목을 완벽히 가져온다.

## 1단계: 공식 사이트 데이터 수집 (반드시 먼저)

### 1-1. 제품 상단 정보 수집
공식 사이트 `https://www.lxzin.com/zin/product/{제품ID}` 를 **모든 제품에 대해** 방문하여 아래 항목을 수집한다:

| 항목 | 수집 위치 | 예시 |
|------|---------|------|
| **제품코드(model_id)** | 제품코드 필드 | CM22771 |
| **사이즈** | 사이즈 필드 | 1.8mm(T) x 1,830mm(W) x 35m(L) |
| **색상** | 색상 필드 (색상 칩 텍스트) | 베이지, 화이트 |
| **포장단위** | 스펙 테이블 내 포장 단위 | 1Roll / 35M |
| **카테고리** | 상단 breadcrumb | 시트 > 시트 스탠다드 1.8(뉴청맥) |

⚠️ **주의사항**:
- 색상은 반드시 **색상 칩 옆의 텍스트**를 정확히 읽는다 (예: "라이트브라운" ≠ "브라운")
- 복수 색상이 있으면 쉼표로 구분 (예: "브라운, 라이트브라운")
- 사이즈는 **공식 사이트에 표시된 그대로** 사용한다. 임의로 항목을 추가/제거하지 마라
  - 예: 공식 사이트가 `2.2mm(T) × 1,830mm(W)` 이면 **길이(L)를 추가하지 마라**
  - 예: 공식 사이트가 `1.8T X 1,830W X 35m(L)` 이면 **그대로 입력**하라
- `size` 필드의 형식은 **공식 사이트의 표기 형식**을 따른다 (단위, 구분기호 포함)

### 1-2. 하단 상세 이미지/콘텐츠 수집
공식 사이트의 하단 상세 섹션을 **전부 스크롤하면서** 아래 항목을 수집한다:

| 항목 | 설명 |
|------|------|
| **히어로 이미지** | 상단 대표 이미지 URL |
| **특장점 아이콘+텍스트** | 아이콘 이미지 URL + 제목/부제 텍스트 |
| **제품 구조도** | 구조도 이미지 URL + 설명 텍스트 (레이어명 등) |
| **기능 상세 이미지** | 우수한 품질, 다양한 디자인 등 이미지 URL + 설명 |
| **스펙 테이블** | 사이즈, 포장 단위 등 테이블 내용 |
| **인증 & 수상** | 인증 로고 이미지 URL + 설명 텍스트 |
| **주의사항** | 사용 시 / 안전관리 시 / 시공 시 전체 텍스트 |

⚠️ **주의사항**:
- 이미지 URL은 `https://octapi.lxzin.com/...` 형식으로 정확히 복사
- 텍스트는 한 글자도 빠짐없이 복사 (줄바꿈 위치 포함)
- 공식 사이트에 없는 내용은 절대 추가하지 않는다
- 공식 사이트의 내용을 절대 생략하지 않는다

## 2단계: 데이터 입력 (lxzin-products.js)

수집한 데이터를 `src/data/lxzin-products.js`에 입력한다.

### 필수 필드 체크리스트
```
☐ id: 공식 사이트 URL의 제품 ID
☐ title: 제품명 (공식 사이트 제목 — 띄어쓰기 정확히)
☐ model_id: 제품코드 (예: CM22771)
☐ imageUrl: 제품 이미지 경로 (아래 규칙 필수 준수)
☐ specifications.size: 전체 사이즈 문자열
☐ specifications.color: 공식 사이트 색상 (정확히)
☐ price: 사용자가 정한 가격
☐ priceLabel: '판매금액:' 또는 사용자 지정
☐ priceUnit: '/m²' 또는 '' (사용자 지정)
☐ subCategory: 공식 사이트 카테고리
☐ category: 대분류
☐ packaging: 포장단위 정보
```

### 🖼️ imageUrl 규칙 (반드시 준수)

imageUrl은 아래 패턴을 따른다:
```
/assets/lxzin/{model_id의 하이픈→언더스코어}_{제품ID}.jpg
```

**예시:**
| model_id | 제품 ID | imageUrl |
|----------|---------|----------|
| ZJ44351-22 | 100781 | `/assets/lxzin/ZJ44351_22_100781.jpg` |
| CM22771 | 9648 | `/assets/lxzin/CM22771_9648.jpg` |

⚠️ **핵심 주의사항 (과거 버그 방지)**:
- `imageUrl`의 model_code 부분은 반드시 **해당 제품 자신의 model_id**와 일치해야 한다
- 제품 데이터(title, model_id, color 등)를 수정할 때 **imageUrl도 반드시 함께 수정**하라
- 데이터를 밀거나(shift) 재정렬할 때 imageUrl이 밀리지 않도록 **각 제품의 imageUrl이 자신의 model_id와 id를 포함하는지** 반드시 검증하라
- 이미지 파일이 `public/assets/lxzin/` 폴더에 실제 존재하는지 확인하라. 없으면 공식 사이트에서 다운로드하라

## 3단계: Feature 컴포넌트 작성 (하단 이미지)

`src/components/product/` 폴더에 Feature 컴포넌트를 만든다.

### 표준 섹션 구조 (공식 사이트 순서 따름)
```
1. 안내 문구 (제품명 변경 안내 등)
2. 히어로 이미지 + 특장점 카드 (3개 가로 배치)
3. 제품 구조도
4. 기능 상세 (우수한 품질 / 다양한 디자인 등)
5. 스펙 테이블
6. 인증 & 수상
7. 주의사항 (사용 시 → 안전관리 시 → 시공 시 순서)
```

### 표준 스타일 규격
- 전체 컨테이너: `maxWidth: 970px`, `margin: 0 auto`
- 섹션 간격: `marginBottom: 200px`
- 제목 폰트: `fontSize: 16px`, `fontWeight: bold`, `textAlign: center`
- 본문 폰트: `fontSize: 14px` 또는 `16px`
- 이미지: `display: block`, `margin: 0 auto`

## 3-2단계: 장바구니 포장단위 배지 설정

장바구니에서 포장단위가 올바르게 표시되도록 설정한다.

### 포장단위 배지 동작 원리 (`ShoppingCartCheckout.jsx`)

장바구니에서 포장단위 배지는 2가지 방식으로 표시된다:

**1. 에디톤 제품 (박스 배송)**
- `EDITON_BOX_INFO` 상수에 등록된 subCategory와 매칭
- 표시 형식: `📦 박스배송 · {N}매 / 1박스`
- 예: 에디톤 스톤 → `📦 박스배송 · 4매 / 1박스`

**2. 시트/롤 제품 (packaging 필드)**
- `specifications.packaging` 값을 읽어서 배지에 표시
- 표시 형식: `📦 {packaging} 1R`
- 예: 뉴청맥 → `📦 35M 1R`, 은행목 → `📦 30M 1R`

### 필수 작업

새 시트/롤 제품 추가 시 반드시 `specifications.packaging` 필드를 설정한다:

| 제품 라인 | packaging 값 | 장바구니 표시 |
| -------- | ----------- | ---------- |
| 뉴청맥 (스탠다드 1.8) | `'35M'` | 📦 35M 1R |
| 은행목 (스탠다드 2.0) | `'30M'` | 📦 30M 1R |
| 시트 프리미엄 2.2 | 공식 사이트 확인 후 설정 | 📦 {M수} 1R |
| 엑스컴포트 | 공식 사이트 확인 후 설정 | 📦 {M수} 1R |

### 배지 스타일 (통일)
- 배경: `bg-amber-50`
- 텍스트: `text-amber-700`
- 테두리: `border-amber-200`
- 아이콘: 📦

## 4단계: 검증

### 4-1. 데이터 검증

- 전체 제품 수 확인
- 각 제품의 색상/사이즈/제품코드가 공식 사이트와 100% 일치하는지 확인
- 브라우저에서 제품 상세 페이지 열어서 표시 확인

### 4-2. 🖼️ imageUrl ↔ 이미지 파일 매핑 검증 (필수!)

**모든 제품에 대해** 아래 3가지를 반드시 확인한다:

1. **imageUrl 패턴 확인**: `imageUrl`의 model_code 부분이 해당 제품의 `model_id`와 일치하는가?
   ```
   ✅ id:"100781", model_id:"ZJ44351-22", imageUrl:"/assets/lxzin/ZJ44351_22_100781.jpg"
   ❌ id:"100781", model_id:"ZJ44351-22", imageUrl:"/assets/lxzin/ZJ44241_22_100781.jpg"  ← model_code 불일치!
   ```

2. **이미지 파일 존재 확인**: `public/assets/lxzin/` 폴더에 해당 파일이 실제 존재하는가?
   - 없으면 공식 사이트에서 다운로드 필요

3. **이미지 내용 검증**: 브라우저에서 카테고리 페이지를 열고, 각 제품 카드의 이미지가 제품명에 맞는 패턴인지 확인
   - 그레이 콘크리트 → 콘크리트 타일 이미지
   - 로즈 마블 → 마블(대리석) 이미지
   - 오크 → 우드 이미지

⚠️ **과거 발생한 버그**: 제품 데이터(title, model_id)를 수정하면서 imageUrl을 함께 수정하지 않아, 이미지가 1칸씩 밀려 엉뚱한 제품 사진이 표시되었음. **반드시 한 번에 수정하라.**

### 4-3. 하단 이미지 검증

- 공식 사이트와 나란히 비교
- 이미지 누락/깨짐 없는지 확인
- 텍스트 누락/오타 없는지 확인

### 4-4. 장바구니 검증

- 장바구니에 담아서 제품코드 정확히 표시되는지 확인
- 포장단위 배지가 `📦 {M수} 1R` 형식으로 정상 표시되는지 확인
- 배지 색상이 amber 테마(노란색 계열)로 통일되는지 확인
- 수량 증감 시 금액 계산이 올바른지 확인

## 5단계: 보호 등록

작업 완료 후 `editon-protection.md`에 추가:

1. Feature 컴포넌트 파일명 → 보호 대상 파일 테이블에 추가
2. 확정된 제품 데이터 (가격, 사이즈, 색상 등) → 확정 데이터 섹션에 추가
3. 제품 목록 테이블 → ID, 제품명, 제품코드, 색상 기록
4. "✅ 작업 완료" 표시

---

## ⛔ 절대 하지 말 것

1. 공식 사이트 **확인 없이** 색상/사이즈를 추측하여 입력하지 마라
2. 하단 이미지의 텍스트를 **요약하거나 생략**하지 마라
3. 공식 사이트에 없는 **임의의 내용을 추가**하지 마라
4. 한 제품만 확인하고 **나머지는 동일할 것이라 가정**하지 마라
5. 작업 완료 후 **보호 등록을 빼먹지** 마라
6. title/model_id를 수정할 때 **imageUrl을 함께 수정하지 않는 것**은 절대 금지
7. imageUrl의 model_code가 **다른 제품의 model_id를 참조하는 상태**를 방치하지 마라
